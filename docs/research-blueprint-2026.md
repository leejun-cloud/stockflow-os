# 스톡 사진·영상 자동 업로드 SaaS — 리서치 종합 & 설계 청사진 (2026)

> 12개+ 병렬 리서치(11개 플랫폼 + 신규기관 + 로그인자동화 + 통합프로필 + 수익집계 + 수익예측 + 대체수익모델 + AI데이터) 종합.
> 방향 확정: **본인 계정 기반 반자동 툴(Model B) + 사진·영상 동시 지원**.

---

## 0. 핵심 전략 결론

- **대리판매(Aggregator, Wirestock식) 모델은 죽었다.** Wirestock이 2026년 1월 스톡 배포 사업 철수 → 플랫폼 리스크 실증. 우리는 **사용자가 계정을 소유하고 우리는 자동화만 하는 Model B**로 간다.
- **완전 무인은 정책이 원천 차단한다.** 공개 업로드 API가 한 곳도 없고, 자동 가입·다중계정은 Adobe·Shutterstock이 ToS로 명시 금지. → 업계가 수렴한 유일한 지속가능 형태 = **HITL(Human-in-the-Loop) 반자동**.
- **영상을 1급으로 취급한다.** 영상 다운로드당 수익이 사진의 ~10배(Adobe 사진 $0.66 vs 영상 $14), 등급 상승도 빠름(10 vs 100 다운로드). 사진 파이프라인에 영상 규격/코덱만 추가하면 재사용 가능.

---

## 1. 자동화의 3계층 (합법 경계선)

```
[완전 자동 · 합법]     AI 메타데이터 생성 → IPTC/XMP 임베드 → (S)FTP 병렬 전송
[반자동 · HITL]        온보딩 폼 자동채움 → 사용자가 CAPTCHA/2FA/인증만 클릭 → 세션쿠키 재사용
[사람 필수 · 통제불가] 계정생성 승인 · W-8BEN · 신분증 · 최종 Submit 클릭 · 심사
```

**모든 플랫폼 공통**: 공개 업로드 API 없음 → (S)FTP가 유일 공식 채널. IPTC/XMP 임베드하면 자동 인식. 웹 UI 봇 조작은 ToS 위반 리스크.

## 3. 수익 현실 (제품 명분의 근거)

**1000장 고퀄 포트폴리오, 8개 사이트 분산, 1년차 기준:**

| 시나리오 | 월수익 | 비고 |
|---|---|---|
| 비관 | $30~80 | 신규 노출 저조 |
| 중간(현실) | $100~200 | 상위 2개 사이트가 절반 |
| 낙관 | $250~400 | 니치 적중 |
| 2~3년 성숙 | 위의 1.5~2배 | 누적 효과 |

- **핵심 통찰**: 장당 월수익 $0.01~0.05. 파레토 — 1000장 중 ~200장이 수익의 80%. 8개 사이트 = 8배 아니라 ~2.5~4배.
- **제품 명분**: RPI가 낮으므로 **사용자 개입 시간을 0에 수렴시켜야 ROI 성립**. "그냥 자동으로 굴러가는 부수입"이 우리가 파는 가치.
- **영상 추가 근거**: 영상은 다운로드당 $4~7(사진 10배), 등급 상승 빠름 → 같은 노동으로 수익 상방 확대.

## 4. 시스템 설계

### 4.1 데이터 모델 (통합 마스터 프로필)
`identity` / `address` / `tax` / `payment`는 1회 입력 공통, `agencyAccounts[]`에 사이트별 차이만 오버라이드.
(상세 JSON 스키마는 리서치 원본 참조 — 통합 프로필 스키마 섹션)

**한국 거주자 W-8BEN 자동 프리필 값:**
- Part I-6 Foreign TIN = 주민등록번호(YYMMDD-XXXXXXX)
- Part II-9 조약국 = "Korea, Republic of"
- Part II-10 = **Article 14, 저작권 로열티 세율 10%** (미기재 시 30% 원천징수)
- 지급: 한국은 **Payoneer 사실상 필수**(Adobe 등 PayPal 불가국)

### 4.2 자동화 엔진
1. **메타데이터 생성**: AI 비전(제목/설명/키워드) — RJ-Auto-Metadata(AGPL, 설계만 참고)식
2. **임베드**: exiftool로 IPTC/XMP를 파일에 기록 (사진·영상 공통)
3. **규격 변환**: 사이트별 최소해상도/키워드캡(Adobe 49·나머지 50)/카테고리 매핑/코덱 자동
4. **다중 (S)FTP 팬아웃**: uploadfun(MIT)식 병렬+재시도+검증 워커. 사진은 FTP, 영상은 사이트별 전용 엔드포인트
5. **온보딩 위저드(HITL)**: 폼 자동채움 → CAPTCHA/2FA/인증은 사용자에게 넘김 → 세션쿠키 저장
6. **최종 제출**: 세션쿠키 재사용 시도(eugene-ga/shutterstock식, 리스크 고지), 안 되면 체크리스트 안내

### 4.3 수익 대시보드 (3-Tier 하이브리드)
- **Tier 1 (공식 API)**: 현재 비어있음(수익 API 제공하는 곳 0). 생기면 편입
- **Tier 2 (세션쿠키 스크래핑)**: 안티봇 약한 곳(Alamy/Dreamstime/123RF) 옵트인
- **Tier 3 (CSV 임포트) — 기본 경로**: 사용자가 대시보드 CSV 다운로드 → 업로드 → 파싱
- **핵심 지표**: 총수익, 사이트별, **RPD(다운로드당 수익)**, 베스트셀러, 통화·지급주기 정규화(기존 도구 빈틈)

## 5. 재사용 오픈소스 (설계 벤치마크)
1. **RJ-Auto-Metadata** (52★, AGPL — 설계만): AI메타 + XMP + 플랫폼별 CSV
2. **uploadfun** (MIT): 다중 (S)FTP 팬아웃 워커
3. **eugene-ga/shutterstock** (MIT, TS): 세션쿠키 기반 제출 자동화
- FTP credential 암호화·업로드 전 검증 UX는 Xpiks(GPL) 기능목록 참고

## 6. 법적/리스크 가드레일
- **1인 1계정 원칙 준수** — 다중계정 자동생성 절대 금지(정지 1순위)
- 가상번호·임시메일·CAPTCHA 솔버 **사용 안 함**
- credential: 원문 비번 미저장, **세션토큰 암호화 + 짧은 TTL**
- 스크래핑은 본인 계정 데이터 한정, 대량 다운로드와 분리
- 사용자에게 ToS 리스크 명시 고지
- Pond5·Canva·500px는 **AI 생성물 금지** — 업로드 라우팅에서 필터

---

## 7. MVP 범위 (권장)
1. AI 메타데이터 생성 + IPTC/XMP 임베드 (사진·영상)
2. P0 3사(Alamy·Adobe·Shutterstock) 다중 (S)FTP 팬아웃
3. 온보딩 위저드 + W-8BEN 자동 프리필(한국)
4. CSV 임포트 수익 대시보드
5. 반려 예방 사전검사(해상도/초점/AI표시)

**타깃 재조정**: "왕초보"보다 "사진은 있는데 여러 사이트 올리기 귀찮아 1~2곳만 쓰는 사람"이 전환율 높음.
